The idea

Security review that keeps up with shipping.

The problem

Scanner findings stay scattered across tools, and there's no enforced checkpoint between development, security, and deployment — so risky code reaches production.

The solution

Every push is auto-scanned, findings are unified into one dashboard, and only Security-approved commits can be deployed — every decision recorded and auditable.

Architecture

Pipeline & scanning → portal & governance.

A Gitea push fires a Jenkins pipeline that builds the code and runs all three scanners. Results are POSTed to the portal's REST API and written to MySQL. The Security team reviews findings and approves or rejects each commit; only approved commits reach the Operations deployment queue.
